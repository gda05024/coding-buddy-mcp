# Coding Buddy MCP 테스트 케이스

아무 프로젝트에서 **새 세션**을 열고 아래 프롬프트를 하나씩 입력합니다.
각 테스트 후 `/clear`로 초기화하고 다음 테스트를 진행하세요.

---

## 테스트 0: 연결 확인
```
/mcp
```
- [ ] coding-buddy가 목록에 보인다

---

## A. 구체성 강제 (모호한 요청 → 되물어야 함)

### A-1. 모호한 버그 요청
```
버그 찾아줘
```
- [ ] 도구를 호출하지 않는다
- [ ] 파일 경로/증상을 되묻는다

### A-2. 모호한 리팩토링
```
코드 리팩토링 해줘
```
- [ ] 어떤 파일/범위인지 되묻는다

### A-3. 모호한 테스트 요청
```
테스트 작성해줘
```
- [ ] 어떤 파일/함수인지 되묻는다

### A-4. 모호한 에러 보고
```
에러가 나
```
- [ ] 에러 메시지와 파일을 물어본다

### A-5. 구체적 요청 → 바로 실행
```
package.json의 dependencies 보여줘
```
- [ ] 되묻지 않고 바로 파일을 읽는다
- [ ] 💡 모델 추천이 첫 줄에 나온다

---

## B. 모델 추천

### B-1. 단순 작업 → Haiku
```
README.md에서 오타 수정해줘. recieve를 receive로
```
- [ ] 💡 모델 추천: Haiku ($1/$5)

### B-2. 중간 작업 → Sonnet
```
src/components/LoginForm.tsx에 비밀번호 유효성 검사 추가해줘
```
- [ ] 💡 모델 추천: Sonnet ($15/$75)

### B-3. 복잡한 작업 → Opus + Plan Mode
```
인증 시스템을 JWT에서 세션 기반으로 전체 마이그레이션 해줘
```
- [ ] 💡 모델 추천: Opus ($15/$75)
- [ ] Plan Mode를 먼저 쓰라고 제안
